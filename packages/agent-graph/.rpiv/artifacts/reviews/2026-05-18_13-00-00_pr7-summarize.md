### Q1 — Error code semantic mismatch

**Severity:** 🟡 → Verified

### Q2 — Dropped reconciliation warnings in summarize CLI handler

**Severity:** 🟡 → Verified
**Files:** `src/cli.rs:438-453`

**Finding:** The `summarize` CLI handler used `ResponseEnvelope::ok(...)` which drops `result.warnings`. Other CLI handlers (e.g., `status`) preserved warnings via `ok_with_warnings(...)`.

**Fix:** Changed to `ok_with_warnings(...)` and implemented `Display` for `ReconciliationWarning` to enable canonical formatting.

**Test added:** `summarize_propagates_desync_warning` verifies warning propagation.

### Q3 — Predicate coherence on Status enum

**Severity:** 🟡 → Verified
**Files:** `src/reconcile.rs:984`, `src/reconcile.rs:1003`, `src/reconcile.rs:1035`

**Finding:** Three separate `matches!(status, Status::Completed | Status::Skipped)` conditionals exist across `reconcile.rs`. The pattern is internally consistent but should be extracted into a helper when a fourth usage appears.

**No action taken:** Only two unique predicates (dependency-readiness vs. summary-eligibility). Refactoring deferred until pattern recurs.
