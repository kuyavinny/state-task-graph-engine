## Detailed Findings

### I1 🟡 — Partial persistence safety (FIXED)

**File:** `src/reconcile.rs`  
**Original:** `io::write_graph(...)` then `io::append_events_batch(...)`  
**Risk:** If `append_events_batch` fails (disk full, permissions), the graph YAML is updated but events are missing. The graph revision advances but events cannot be replayed.

**Fix:** Swapped order — events written first, then graph. If graph write fails, the next reconcile detects the desync (events exist but graph lags) and the system recovers. Commit `0076881`.

### I2 🟡 — No CLI desync integration test (FIXED)

**File:** `tests/reconcile.rs`  
**Finding:** Unit test covered desync rejection, but CLI-level path was untested.

**Fix:** Added `append_nodes_desync_rejected` — writes graph with revision 5 but empty event log, runs `stg append-nodes`, asserts `EVENT_LOG_DESYNC` error code. Commit `0076881`.

### I3 🟡 — Missing edge case tests (PARTIALLY FIXED)

**Finding:** No integration tests for empty node file, non-existent `--file`, duplicate IDs in YAML, unknown dependency, invalid YAML, missing required fields.

**Fix:**
- Added `append_nodes_file_not_found` — verifies `FILE_NOT_FOUND` for missing `--file` path (commit `0076881`)
- Added `append_nodes_empty_list_returns_ok_with_no_events` — unit test for empty input (commit `0076881`)
- **Deferred:** empty YAML file, unknown dependency, invalid YAML, missing fields — lower risk, can be incremental

### I4 🟡 — Generic IO error for missing file (FIXED)

**File:** `src/cli.rs`, `src/error.rs`, `src/model.rs`  
**Original:** `std::fs::read_to_string(&file)?` returned generic IO_ERROR.

**Fix:** Added `AppError::FileNotFound { path: String }` with `ErrorCode::FileNotFound`. CLI matches `std::io::ErrorKind::NotFound` and returns the structured error with the file path in details. Commit `0076881`.

### S1 🔵 — Misleading turbofish (FIXED)

**File:** `src/main.rs`  
**Original:** `ResponseEnvelope::<()>::from_error(&e, None)` — the `()` turbofish is meaningless because `from_error` returns `ResponseEnvelope<serde_json::Value>`.

**Fix:** Changed to `ResponseEnvelope::<serde_json::Value>::from_error(...)`. Commit `0076881`.

### S2 🔵 — Incomplete lease normalization (FIXED)

**File:** `src/reconcile.rs`  
**Original:** `if node.lease.claimed_by.is_none() { node.lease = Lease::empty(); }`  
**Risk:** Partial leases (e.g., `claimed_by: None` but `claimed_at: Some(...)`) silently lose orphan timestamps.

**Fix:** Changed to check all three fields (`claimed_by`, `claimed_at`, `expires_at`) are None before normalizing. Partial lease data passes through to validation which catches it. Commit `0076881`.

### S3 🔵 — Vague event revision comment (FIXED)

**File:** `src/reconcile.rs`  
**Original:** `// Use tentative revision; will be finalized at persist` with `// Index in events array will be resolved at persist time`

**Fix:** Replaced with `// Tentative revision: indexed from 1 relative to graph_revision.` and `// Final revision = old_rev + total_event_count is computed at persist time below.` Commit `0076881`.

### S4 🔵 — No post-merge validation test (DEFERRED)

**File:** `src/reconcile.rs` (unit tests)  
**Finding:** No test for validation failures after merge beyond cycle detection (e.g., appending a node with a non-existent dependency).

**Recommendation:** Add `append_nodes_rejects_unknown_dependency` — appends a node depending on a non-existent ID, asserts `GraphValidationFailed`.

**Deferred:** Lower-risk edge case; cycle detection covers the same code path.

### S5 🔵 — Post-merge integration test gaps (DEFERRED)

**File:** `tests/reconcile.rs`  
**Finding:** No integration tests for empty input YAML, invalid YAML syntax, duplicate IDs in YAML, unknown dependency across nodes.

**Recommendation:** Add each scenario as a separate test.

**Deferred:** CLI-level error handling works correctly for these cases (returns structured errors); can be added incrementally.
